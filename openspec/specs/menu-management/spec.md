# menu-management Specification

## Purpose

菜单管理负责存储前端导航元数据、返回菜单树、同步前端路由，并控制角色和用户可见菜单。它定义菜单记录维护、角色菜单分配、用户初始化上下文中菜单树的生成规则。

## Requirements

### Requirement: 菜单树查询
系统 SHALL 按排序和 ID 返回树形菜单。

#### Scenario: 管理员查询菜单
- **WHEN** 管理员调用菜单列表接口
- **THEN** 系统返回按 `parent_id` 组装的所有菜单
- **AND** 子菜单嵌套在父菜单的 `children` 中

### Requirement: 菜单 CRUD
系统 SHALL 允许管理员创建、修改和删除菜单记录。

#### Scenario: 创建菜单
- **WHEN** 管理员提交菜单名称，以及可选的路径、组件、图标、父级、排序、类型和状态
- **THEN** 系统创建菜单
- **AND** 当类型或状态未提供或无效时，系统使用默认值

#### Scenario: 菜单路径冲突
- **WHEN** 创建或修改请求使用了已被其他菜单占用的路径
- **THEN** 系统拒绝请求

#### Scenario: 父级菜单指向自己
- **WHEN** 修改请求将 `parent_id` 设置为当前菜单自身 ID
- **THEN** 系统拒绝请求

#### Scenario: 修改菜单
- **WHEN** 管理员提交有效的菜单变更字段
- **THEN** 系统只更新提交的字段
- **AND** 如果没有可更新字段，系统拒绝请求

#### Scenario: 删除存在子菜单的菜单
- **WHEN** 管理员删除仍有子菜单的菜单
- **THEN** 系统拒绝删除

#### Scenario: 删除叶子菜单
- **WHEN** 管理员删除没有子菜单的菜单
- **THEN** 系统删除该菜单对应的角色菜单关联
- **AND** 系统硬删除菜单记录

### Requirement: 角色菜单分配
系统 SHALL 允许管理员替换某个角色拥有的菜单列表。

#### Scenario: 分配菜单到角色
- **WHEN** 管理员提交角色 ID 和菜单 ID 列表
- **THEN** 系统清除该角色已有的角色菜单记录
- **AND** 系统插入本次提交的关联记录

#### Scenario: 查询角色菜单
- **WHEN** 管理员查询某个角色拥有的菜单
- **THEN** 系统返回该角色已分配且启用的菜单树
- **AND** 如果没有菜单，系统返回空列表

### Requirement: 用户菜单树
系统 SHALL 从用户拥有的所有角色中推导该用户可见的菜单树。

#### Scenario: 用户拥有角色菜单
- **WHEN** 系统构建用户初始化上下文
- **THEN** 系统收集用户所有角色关联的菜单 ID
- **AND** 系统对菜单 ID 去重
- **AND** 系统返回启用菜单组成的树

#### Scenario: 用户没有分配菜单
- **WHEN** 用户的角色没有任何角色菜单关联
- **THEN** 系统返回空菜单列表

### Requirement: 前端路由同步菜单
系统 SHALL 能够根据前端路由元数据创建缺失的菜单记录。

#### Scenario: 同步创建新菜单
- **WHEN** 管理员向同步接口提交路由元数据
- **THEN** 系统为不存在的路径创建菜单记录
- **AND** 如果存在 `parent_path`，系统根据它解析父菜单
- **AND** 系统返回本次创建的菜单数量

#### Scenario: 同步遇到已存在路径
- **WHEN** 提交的路由路径已存在于菜单表
- **THEN** 系统跳过该条数据
