# add-zap-runtime-logging

## Why

当前项目中启动日志、初始化失败日志、文件轮转日志分散使用 `fmt` 和标准库 `log`。这会导致日志格式不统一，也不方便后续接入文件切割、生产环境 JSON 日志和集中式日志系统。

## What Changes

- 增加 Zap 运行日志模块。
- 增加日志配置项，支持日志级别、格式、输出文件和文件轮转。
- 增加全局 `global.Logger`。
- 在启动链路中初始化日志模块。
- 将 MySQL、Redis、MinIO 初始化日志接入 Zap。
- 将文件轮转任务日志接入 Zap。
- 增加 Gin 请求运行日志中间件，仅记录请求元信息，不记录请求体。

## Out of Scope

- 不实现 API 审计日志入库。
- 不实现登录日志、操作日志、权限变更日志、数据访问日志查询接口。
- 不记录密码、token、验证码、请求体等敏感数据。
