# 字典管理文档

## 当前状态

> **状态**: 已实现基础 CRUD。

字典管理用于统一维护系统中的可配置枚举，避免把用户状态、文件类型、消息类型等固定选项写死在代码里。

- [x] 修改字典类型
- [x] 删除字典类型
- [x] 字典条目列表
- [x] 创建字典条目

## 数据模型

### 字典类型

表名：`dict_types`

```go
type DictType struct {
    ID     uint
    Name   string
    Code   string
    Remark string
    Sort   int
    Status int
}
```

示例：

```text
Name: 用户状态
Code: user_status
```

### 字典条目

表名：`dict_items`

```go
type DictItem struct {
    ID       uint
    TypeCode string
    Label    string
    Value    string
    Remark   string
    Sort     int
    Status   int
}
```

示例：

```text
TypeCode: user_status
Label: 启用
Value: 1
```

## 接口列表

### 1. 获取字典类型列表

```http
GET /api/admin/dict-types?page=1&size=10&keyword=user&status=1
Authorization: Bearer <admin_token>
```

### 2. 创建字典类型

```http
POST /api/admin/dict-types
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "name": "用户状态",
  "code": "user_status",
  "remark": "用户启用状态",
  "sort": 1,
  "status": 1
}
```

### 3. 修改字典类型

```http
PUT /api/admin/dict-types/1
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "name": "用户状态",
  "remark": "用户账号状态",
  "sort": 1,
  "status": 1
}
```

### 4. 删除字典类型

```http
DELETE /api/admin/dict-types/1
Authorization: Bearer <admin_token>
```

删除字典类型时，会同步删除该类型下的字典条目。

### 5. 获取字典条目列表

```http
GET /api/admin/dict-items?page=1&size=10&type_code=user_status&keyword=启用&status=1
Authorization: Bearer <admin_token>
```

### 6. 创建字典条目

```http
POST /api/admin/dict-items
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "type_code": "user_status",
  "label": "启用",
  "value": "1",
  "remark": "正常可用",
  "sort": 1,
  "status": 1
}
```

### 7. 修改字典条目

```http
PUT /api/admin/dict-items/1
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "label": "启用",
  "value": "1",
  "remark": "账号正常",
  "sort": 1,
  "status": 1
}
```

### 8. 删除字典条目

```http
DELETE /api/admin/dict-items/1
Authorization: Bearer <admin_token>
```

### 9. 按类型编码获取启用条目

```http
GET /api/dicts/user_status/items
```

返回：

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "type_code": "user_status",
      "label": "启用",
      "value": "1",
      "remark": "正常可用",
      "sort": 1,
      "status": 1
    }
  ]
}
```

## 约束说明

- 字典类型 `code` 唯一。
- 同一个 `type_code` 下，字典条目 `value` 唯一。
- 删除字典类型时，会硬删除对应字典条目。
- 修改字典类型 `code` 时，会同步更新该类型下所有字典条目的 `type_code`。
- 前端读取接口只返回启用的字典类型和启用的字典条目。

## Apifox 测试顺序

1. 创建字典类型 `user_status`。
2. 创建字典条目：`1=启用`、`0=禁用`。
3. 查询字典类型列表。
4. 查询字典条目列表。
5. 调用 `GET /api/dicts/user_status/items`，确认只返回启用条目。
6. 修改字典条目状态为禁用，再次查询前端读取接口。
7. 删除字典类型，确认对应条目被同步删除。
