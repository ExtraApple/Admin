# 配置管理

## 当前方案

项目使用 `config.yaml + .env` 混合配置：

- `config.yaml` 保存配置结构、端口、开关、默认非敏感值。
- `.env` 保存密码、密钥等敏感值。
- `.env.example` 保存环境变量模板，可以提交到 Git。
- `.env` 已加入 `.gitignore`，不要提交真实密码。

## 启动时加载顺序

```text
读取 .env（如果存在）
读取 config.yaml
根据 *_env 字段读取环境变量
得到最终 Config
```

如果 `config.yaml` 中配置了 `password_env`、`secret_env` 等字段，但对应环境变量不存在，服务会启动失败。

## 本地 .env 示例

```env
MYSQL_PASSWORD=000000
REDIS_PASSWORD=
MINIO_USERNAME=minioadmin
MINIO_PASSWORD=minioadmin
JWT_SECRET=admin-jwt-secret-key-2024
ADMIN_PASSWORD=Admin@123456
```

`REDIS_PASSWORD=` 可以为空，表示本地 Redis 无密码。

## config.yaml 示例

```yaml
mysql:
  host: 127.0.0.1
  port: 3306
  user: root
  password_env: MYSQL_PASSWORD
  db: admin

jwt:
  secret_env: JWT_SECRET
  expire: 15
  refresh_expire: 10080

admin:
  username: admin
  password_env: ADMIN_PASSWORD
  email: admin@test.com
  nickname: 超级管理员
```

## 超级管理员初始化

服务启动并完成数据库迁移后，会检查新 RBAC 体系中是否存在启用的超级管理员：

```text
users
  -> user_roles
  -> roles.code = "admin"
```

当前不再依赖 `users.role = "admin"` 判断管理员身份。

初始化规则：

- 如果 `roles.code = admin` 不存在，自动创建。
- 如果已经存在启用的 admin 角色用户，不修改任何账号和密码。
- 如果不存在启用的 admin 角色用户，使用 `config.yaml + .env` 中的 `admin` 配置创建超级管理员。
- 如果配置中的管理员用户名已经存在，但没有绑定 admin 角色，服务启动失败，避免误把普通用户提升为超级管理员。
- 如果配置中的管理员用户已经绑定 admin 角色但被禁用或软删除，会恢复启用并重置为当前 `ADMIN_PASSWORD`。

## 部署环境变量

本地开发可以使用 `.env`。Linux 服务器、Docker、Kubernetes 不一定需要 `.env` 文件，只要在进程启动前注入同名环境变量即可：

```text
MYSQL_PASSWORD
REDIS_PASSWORD
MINIO_USERNAME
MINIO_PASSWORD
JWT_SECRET
ADMIN_PASSWORD
```

其中 `REDIS_PASSWORD` 允许为空，其他变量不能为空。生产环境建议使用 Docker Secret、Kubernetes Secret、CI/CD Secrets 或云厂商 Secret Manager 管理这些值。


### 注：如下版本还未测试。
### Linux 直接运行

临时启动可以在当前 shell 中导出环境变量：

```bash
export MYSQL_PASSWORD='your-mysql-password'
export REDIS_PASSWORD=''
export MINIO_USERNAME='minioadmin'
export MINIO_PASSWORD='your-minio-password'
export JWT_SECRET='your-jwt-secret'
export ADMIN_PASSWORD='your-admin-password'

./admin
```

使用 systemd 部署时，建议把敏感变量放到独立的环境文件中：

```ini
# /etc/admin/admin.env
MYSQL_PASSWORD=your-mysql-password
REDIS_PASSWORD=
MINIO_USERNAME=minioadmin
MINIO_PASSWORD=your-minio-password
JWT_SECRET=your-jwt-secret
ADMIN_PASSWORD=your-admin-password
```

```ini
# /etc/systemd/system/admin.service
[Unit]
Description=Admin API Service
After=network.target

[Service]
WorkingDirectory=/opt/admin
ExecStart=/opt/admin/admin
EnvironmentFile=/etc/admin/admin.env
Restart=always

[Install]
WantedBy=multi-user.target
```

更新后执行：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now admin
sudo systemctl status admin
```

### Docker

直接使用 `docker run` 时，可以用 `--env-file` 传入环境变量：

```bash
docker run -d --name admin \
  -p 8080:8080 \
  --env-file .env \
  admin:latest
```

也可以逐项传入：

```bash
docker run -d --name admin \
  -p 8080:8080 \
  -e MYSQL_PASSWORD='your-mysql-password' \
  -e REDIS_PASSWORD='' \
  -e MINIO_USERNAME='minioadmin' \
  -e MINIO_PASSWORD='your-minio-password' \
  -e JWT_SECRET='your-jwt-secret' \
  -e ADMIN_PASSWORD='your-admin-password' \
  admin:latest
```

`docker-compose.yml` 示例：

```yaml
services:
  admin:
    image: admin:latest
    ports:
      - "8080:8080"
    env_file:
      - .env
```

如果 MySQL、Redis、MinIO 也运行在 Docker 网络中，需要同步修改 `config.yaml` 中的 `mysql.host`、`redis.host`、`minio.host` 为对应服务名。

### Kubernetes

Kubernetes 中建议使用 Secret 注入敏感环境变量：

```bash
kubectl create secret generic admin-env \
  --from-literal=MYSQL_PASSWORD='your-mysql-password' \
  --from-literal=REDIS_PASSWORD='' \
  --from-literal=MINIO_USERNAME='minioadmin' \
  --from-literal=MINIO_PASSWORD='your-minio-password' \
  --from-literal=JWT_SECRET='your-jwt-secret' \
  --from-literal=ADMIN_PASSWORD='your-admin-password'
```

Deployment 中通过 `envFrom` 引用：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: admin
spec:
  replicas: 1
  selector:
    matchLabels:
      app: admin
  template:
    metadata:
      labels:
        app: admin
    spec:
      containers:
        - name: admin
          image: admin:latest
          ports:
            - containerPort: 8080
          envFrom:
            - secretRef:
                name: admin-env
```

如果集群内的 MySQL、Redis、MinIO 使用 Service 暴露，需要同步修改 `config.yaml` 中的 host 为对应 Service 名称，或在部署镜像时挂载不同环境的 `config.yaml`。

### 验证环境变量是否生效

启动成功后，日志中应能看到配置加载、数据库初始化和超级管理员检查完成：

```text
config loaded
mysql initialized
super admin checked
server started
```

如果缺少必填环境变量，服务会在启动阶段失败，并提示类似：

```text
load env config failed: mysql.password_env: MYSQL_PASSWORD is not set
```
