# 菜单管理

## 模块定位

菜单管理维护后台左侧导航、页面入口、按钮权限和角色可见菜单。当前已经接入 API 权限联动，按钮菜单可以绑定后端 API，让前端按钮显示和后端接口鉴权保持一致。

## 已完成功能

- 获取菜单树。
- 创建菜单。
- 修改菜单。
- 删除菜单。
- 同步前端路由到菜单表。
- 角色绑定菜单。
- 获取角色菜单。
- 菜单绑定权限码。
- 按钮权限控制字段。
- 菜单绑定 API。
- 查询菜单绑定 API。

## 菜单类型

```text
type=1 目录
type=2 页面菜单
type=3 按钮菜单
```

建议：

- 目录通常不填 `permission_code`。
- 页面菜单可以绑定页面级权限码。
- 按钮菜单建议绑定操作级权限码，并通过 `menu_apis` 绑定对应 API。

## 接口

所有接口都在管理员路由组下：

```text
/api/admin
```

请求头：

```http
Authorization: Bearer <admin_token>
```

### 获取菜单树

```http
GET /api/admin/menus
```

成功返回：

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "parent_id": 0,
      "name": "系统管理",
      "path": "/system",
      "component": "Layout",
      "icon": "settings",
      "permission_code": "",
      "sort": 1,
      "type": 1,
      "status": 1,
      "children": []
    }
  ]
}
```

### 创建菜单

```http
POST /api/admin/menus
Content-Type: application/json
```

Body：

```json
{
  "parent_id": 0,
  "name": "系统管理",
  "path": "/system",
  "component": "Layout",
  "icon": "settings",
  "permission_code": "",
  "sort": 1,
  "type": 1,
  "status": 1
}
```

### 修改菜单

```http
PUT /api/admin/menus/:id
Content-Type: application/json
```

Body：

```json
{
  "name": "系统设置",
  "icon": "settings",
  "permission_code": "",
  "sort": 2,
  "status": 1
}
```

### 删除菜单

```http
DELETE /api/admin/menus/:id
```

约束：

- 存在子菜单时不允许直接删除。
- 删除菜单会同步清理 `role_menus`。
- 删除菜单会同步清理 `menu_apis`。

### 同步前端路由

```http
POST /api/admin/menus/sync
Content-Type: application/json
```

Body：

```json
{
  "routes": [
    {
      "name": "用户管理",
      "path": "/system/users",
      "component": "system/users/index",
      "icon": "users",
      "permission_code": "admin.users.get",
      "parent_path": "/system",
      "sort": 1,
      "type": 2
    }
  ]
}
```

### 角色绑定菜单

```http
POST /api/admin/roles/:id/menus
Content-Type: application/json
```

Body：

```json
{
  "menu_ids": [1, 2, 3]
}
```

说明：

- `:id` 是角色 ID。
- `menu_ids` 是菜单 ID 数组。
- 该接口是全量替换，会先清空旧菜单，再写入新菜单。

### 获取角色菜单

```http
GET /api/admin/roles/:id/menus
```

返回该角色已经绑定且启用的菜单树。

### 菜单绑定 API

```http
POST /api/admin/menus/:id/apis
Content-Type: application/json
```

Body：

```json
{
  "api_ids": [21, 22],
  "permission_code": "admin.users.save"
}
```

说明：

- `:id` 是菜单 ID。
- `api_ids` 是 API ID 数组。
- `permission_code` 可选。
- 如果不传，系统会从 API 自动推导。
- 多个 API 权限码不一致时，需要手动传 `permission_code`。

### 查询菜单绑定 API

```http
GET /api/admin/menus/:id/apis
```

成功返回：

```json
{
  "code": 200,
  "data": [
    {
      "id": 21,
      "name": "删除用户",
      "method": "DELETE",
      "path": "/api/admin/users/:id",
      "group": "user",
      "permission_code": "admin.users.id.delete",
      "status": 1,
      "need_auth": 1,
      "need_audit": 1
    }
  ]
}
```

## 权限过滤规则

`/api/user/context` 返回当前用户可见菜单时：

- `admin` 角色返回全部启用菜单。
- 普通用户只返回角色绑定菜单。
- 菜单 `permission_code` 为空时，只要角色绑定菜单即可显示。
- 菜单 `permission_code` 不为空时，用户必须拥有对应权限码。
- 子菜单或按钮可见时，会保留父级目录，保证菜单树完整。

## 测试顺序

1. 创建目录菜单。
2. 创建页面菜单。
3. 同步 API 和权限码。
4. 从 API 生成按钮菜单，或手动创建按钮菜单。
5. 菜单绑定 API。
6. 角色绑定菜单。
7. 角色绑定权限。
8. 测试用户重新登录。
9. 调用 `/api/user/context` 检查菜单和权限。
