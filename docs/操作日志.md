操作日志是后台审计基础设施，设计为一层 Gin 中间件，拦截所有 `/api/` 请求，自动记录操作轨迹。

## 模块目标

操作日志负责自动记录每一次 API 调用的完整信息：**谁、什么时候、调了哪个接口、什么参数、返回了什么、耗时多久**。

最终效果：

```text
任意 /api/* 请求
  → AuditLog 中间件拦截
  → 读取请求体（拷贝恢复）
  → c.Next() 执行业务逻辑
  → 异步 goroutine 写入 audit_logs 表
```

## 中间件设计

### middleware/audit_log.go

```go
func AuditLog() gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()

        // 读取请求体（需要拷贝，Gin 的 Body 只能读一次）
        bodyBytes, _ := io.ReadAll(c.Request.Body)
        c.Request.Body = io.NopCloser(bytes.NewBuffer(bodyBytes)) // 恢复给后续 handler

        c.Next() // 执行后续中间件和 handler

        // 异步写入日志，不阻塞用户请求
        go func() {
            // 限制 Body 大小，避免日志膨胀
            bodyStr := string(bodyBytes)
            if len(bodyStr) > 2000 {
                bodyStr = bodyStr[:2000]
            }

            log := model.AuditLog{
                UserID:   c.GetUint("userID"),
                Username: c.GetString("username"),
                Method:   c.Request.Method,
                Path:     c.Request.URL.Path,
                Query:    c.Request.URL.RawQuery,
                Body:     bodyStr,
                Status:   c.Writer.Status(),
                Duration: time.Since(start).Milliseconds(),
                ClientIP: c.ClientIP(),
            }
            global.DB.Create(&log)
        }()
    }
}
```

### 挂载位置


## 日志查询接口设计

参照五类审计日志，后续可提供管理端查询：

### 1. API 调用日志

```http
GET /api/admin/audit-logs?page=1&size=10&user_id=3&method=POST&path=/api/admin/users&start_time=2026-01-01&end_time=2026-06-30
Authorization: Bearer <admin_token>
```

返回所有请求记录，支持按时间、用户、接口筛选。

### 2. 登录日志

```http
GET /api/admin/login-logs?page=1&size=10
Authorization: Bearer <admin_token>
```

筛选 `path = /api/login` 的记录，记录成功/失败状态。

### 3. 操作日志

```http
GET /api/admin/operation-logs?page=1&size=10
Authorization: Bearer <admin_token>
```

筛选 `method IN (POST, PUT, DELETE)` 的记录，关注增删改类操作。

### 4. 权限变更日志

```http
GET /api/admin/permission-logs?page=1&size=10
Authorization: Bearer <admin_token>
```

筛选角色分配/权限变更相关接口的记录。

### 5. 数据访问日志

```http
GET /api/admin/data-access-logs?page=1&size=10
Authorization: Bearer <admin_token>
```

筛选 `method = GET` 且涉及敏感数据读取的记录。

## 安全与性能

| 策略 | 说明 |
|---|---|
| 异步写入 | `go func()` 不阻塞用户请求 |
| Body 截断 | 超过 2000 字符自动截断，防止日志膨胀 |
| 不记录敏感字段 | 密码等敏感字段在写入前过滤/脱敏 |
| 索引优化 | `created_at`、`user_id`、`path` 建索引确保查询速度 |
| 可选开启 | 通过配置或环境变量控制是否记录 |

## 敏感字段过滤

```go
func sanitizeBody(path string, body []byte) []byte {
    // 登录/注册/修改密码接口：脱敏 password 字段
    if strings.Contains(path, "/login") ||
       strings.Contains(path, "/register") ||
       strings.Contains(path, "/password") {
        // 将 password 字段替换为 "***"
        return maskPassword(body)
    }
    return body
}
```

## 分表策略（可选，长期优化）

数据量小时（< 1000万行）单表足够。日志量大的时候按月分表：

```sql
-- 按月创建分表
CREATE TABLE audit_logs_202606 LIKE audit_logs;
CREATE TABLE audit_logs_202607 LIKE audit_logs;
```

或使用 MySQL 分区表：

```sql
ALTER TABLE audit_logs
PARTITION BY RANGE (TO_DAYS(created_at)) (
    PARTITION p202606 VALUES LESS THAN (TO_DAYS('2026-07-01')),
    PARTITION p202607 VALUES LESS THAN (TO_DAYS('2026-08-01'))
);
```

## 数据流

```text
用户请求 /api/admin/users
  → AuditLog 中间件
    ├── 记录 startTime
    ├── 读取并拷贝 Request.Body
    ├── c.Next() 执行业务
    └── goroutine 异步：
        ├── 获取 userID / username（从 JWT Context）
        ├── 获取 method / path / query / clientIP
        ├── 获取 HTTP status code
        ├── 计算 duration = now - startTime
        ├── 截断 body（max 2000 chars）
        ├── 过滤敏感字段
        └── INSERT INTO audit_logs
```

## 注意事项

- 中间件通过 `c.GetUint("userID")` 获取用户信息，依赖 JWTAuth 中间件先执行
- 公开路由（如 `/api/login`）的 `userID` 为 0，username 为空
- Body 读取后必须用 `io.NopCloser` 恢复，否则后续 handler 无法读取
- `go func()` 中不能直接使用 `c *gin.Context`，需先提取所需字段的值拷贝
- 异步写入失败不应影响业务响应（静默失败或仅 log 输出）
- 建议增加配置开关控制中间件启用/禁用
