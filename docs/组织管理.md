# 组织管理文档

## 当前状态

> **状态**: 已实现基础版 — 支持组织单位 CRUD 和组织树。

组织管理用于维护部门、团队、公司层级等组织单位，是后续成员归属、数据权限和组织维度筛选的基础。

## 已实现功能

- [x] 组织单位列表
- [x] 创建组织单位
- [x] 修改组织单位
- [x] 删除组织单位
- [x] 组织树
- [x] 成员绑定组织

## 数据模型

表名：`organizations`

```go
type Organization struct {
    ID       uint
    ParentID uint
    Name     string
    Code     string
    Remark   string
    Sort     int
    Status   int
}
```

成员关联表：`user_organizations`

```go
type UserOrganization struct {
    UserID         uint
    OrganizationID uint
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| `parent_id` | 父组织 ID，0 表示根组织 |
| `name` | 组织名称 |
| `code` | 组织编码，全局唯一 |
| `remark` | 备注 |
| `sort` | 排序 |
| `status` | 状态，1 启用，0 禁用 |

## 接口列表

所有接口都需要管理员 token。

### 1. 获取组织列表

```http
GET /api/admin/organizations?page=1&size=10&keyword=技术&status=1
Authorization: Bearer <admin_token>
```

可选查询参数：

| 参数 | 示例 | 说明 |
|---|---|---|
| `page` | `1` | 页码 |
| `size` | `10` | 每页数量 |
| `keyword` | `技术` | 按名称或编码模糊查询 |
| `status` | `1` | 状态筛选 |

### 2. 获取组织树

```http
GET /api/admin/organizations/tree
Authorization: Bearer <admin_token>
```

返回：

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "parent_id": 0,
      "name": "总公司",
      "code": "headquarters",
      "remark": "",
      "sort": 1,
      "status": 1,
      "children": [
        {
          "id": 2,
          "parent_id": 1,
          "name": "技术部",
          "code": "tech",
          "remark": "",
          "sort": 1,
          "status": 1
        }
      ]
    }
  ]
}
```

### 3. 创建组织

```http
POST /api/admin/organizations
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "parent_id": 0,
  "name": "总公司",
  "code": "headquarters",
  "remark": "根组织",
  "sort": 1,
  "status": 1
}
```

创建子组织仍然使用同一个接口，只需要把 `parent_id` 填成父组织 ID。比如根组织 ID 是 `1`，创建 `技术部 / tech`：

```http
POST /api/admin/organizations
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "parent_id": 1,
  "name": "技术部",
  "code": "tech",
  "remark": "技术部门",
  "sort": 1,
  "status": 1
}
```

### 4. 修改组织

```http
PUT /api/admin/organizations/1
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "parent_id": 0,
  "name": "总公司",
  "remark": "总部",
  "sort": 1,
  "status": 1
}
```

### 5. 删除组织

```http
DELETE /api/admin/organizations/1
Authorization: Bearer <admin_token>
```

删除规则：

- 如果组织存在子组织，拒绝删除。
- 删除组织时会同步删除该组织的成员绑定关系。
- 删除组织使用硬删除，方便后续重新创建相同编码的组织。

### 6. 分配组织成员

```http
POST /api/admin/organizations/1/users
Authorization: Bearer <admin_token>
Content-Type: application/json
```

Body:

```json
{
  "user_ids": [2, 3]
}
```

说明：

- 这个接口是覆盖式分配。
- 传入 `[2, 3]` 后，该组织原有成员会被替换成用户 2 和用户 3。
- 传入空数组 `[]` 可以清空该组织成员。
- 如果传入不存在的用户 ID，会返回错误。

### 7. 查询组织成员

```http
GET /api/admin/organizations/1/users
Authorization: Bearer <admin_token>
```

返回：

```json
{
  "code": 200,
  "data": [
    {
      "id": 2,
      "username": "test",
      "nickname": "测试用户",
      "avatar": "",
      "email": "test@example.com",
      "role": "user",
      "status": 1
    }
  ]
}
```

## 约束说明

- `code` 全局唯一。
- `parent_id = 0` 表示根组织。
- 创建或修改组织时，如果 `parent_id != 0`，父组织必须存在。
- 修改父组织时，不能设置为自己。
- 修改父组织时，不能设置为自己的子组织，避免形成循环树。
- 组织树按 `sort asc, id asc` 排序。
- 一个组织可以绑定多个用户。
- 当前设计允许一个用户被多个组织绑定，后续如果要限制单组织归属，可以在分配时清理用户旧绑定。

## Apifox 测试顺序

1. 创建根组织：`总公司 / headquarters`。
2. 创建子组织：`技术部 / tech`，`parent_id` 填根组织 ID。
3. 查询组织列表。
4. 查询组织树，确认层级正确。
5. 修改子组织名称或排序。
6. 尝试把根组织挂到子组织下面，确认会被拒绝。
7. 尝试删除存在子组织的根组织，确认会被拒绝。
8. 给技术部绑定用户：`POST /api/admin/organizations/:id/users`。
9. 查询技术部成员：`GET /api/admin/organizations/:id/users`。
10. 删除子组织后，再删除根组织。
