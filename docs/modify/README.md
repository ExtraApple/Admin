# modify 修改记录说明

## 目录定位

`docs/modify/` 用来记录项目中已经发生过的重要修改，尤其是跨模块、会影响后续扩展方式的改动。

这里不是正式需求规格，也不是单个接口测试文档。稳定行为以 `openspec/specs/` 为准，模块说明放在 `docs/` 对应模块文档中。

## 适合记录的内容

- 权限链路、认证链路、配置方式这类影响全局架构的修改。
- 涉及多个文件或多个模块的重构。
- 后续需要继续沿用的设计约定。
- 会影响部署、测试、权限同步、数据库补数据的注意事项。
- 已完成但容易忘记的阶段性工作。

## 维护规则

新增修改记录时，建议包含：

```text
修改时间
修改目标
涉及文件
新增接口或新增配置
当前行为
注意事项
验证结果
后续建议
```

如果改动已经成为稳定系统行为，需要同步更新：

```text
openspec/specs/
docs/对应模块文档.md
TODO.md
```

如果只是未来想法，不要直接写成已完成修改，应放入 `TODO.md` 或创建 OpenSpec change。
