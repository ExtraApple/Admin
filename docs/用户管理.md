# 用户管理实施文档

## 当前状态

用户管理是后台系统的核心模块，已完成 **注册、登录、个人信息管理、管理员用户管理** 四大板块。

## 模块目标

用户管理负责用户从注册到登录、从个人信息维护到管理员对用户的增删改查的全流程。

最终效果：

```text
用户访问
  ├── 获取验证码 → 注册账号
  ├── 获取验证码 → 登录 → 获取 JWT
  ├── 携带 JWT → 查看/修改个人信息、修改密码、上传头像
  ├── 携带 JWT → 获取初始上下文（用户+角色+权限+菜单）
  └── 管理员 → 用户列表、修改用户、删除用户、Ban/UnBan
```

## 功能清单

| 功能 | 状态 | 说明 |
|---|---|---|
| JWT 验证 | ✅ | HS256 签名，Access Token 15min / Refresh Token 7天 |
| 验证码登录/注册 | ✅ | 6位数字图片验证码，Redis 5分钟过期，一次性消费 |
| 管理员-用户列表 | ✅ | 分页查询，脱敏返回 |
| 管理员-修改用户 | ✅ | 可改昵称/邮箱/角色/状态，不可改密码 |
| 管理员-删除用户 | ✅ | 软删除（GORM），不可删除自己和其他管理员 |
| 管理员-Ban/UnBan | ✅ | 一键翻转 status 状态 |
| 普通用户-查自己信息 | ✅ | GET /api/user/info |
| 普通用户-修改自己信息 | ✅ | 昵称/头像/邮箱 |
| 普通用户-修改头像 | ✅ | MinIO 存储，uuid 命名，保留最近2个旧文件 |
| 普通用户-修改密码 | ✅ | 旧密码验证 + 两次新密码一致 + 复杂度校验 |
| 登出 | ✅ | Token 加入 Redis 黑名单，过期对齐 Token 有效期 |
| 个人中心-初始上下文 | ✅ | 返回用户信息 + 角色码 + 权限码 + 菜单树 |

## 数据模型设计

### model/user.go

```go
type User struct {
    gorm.Model
    Username string `gorm:"type:varchar(100);not null;uniqueIndex;comment:用户名"`
    Password string `gorm:"type:varchar(255);not null;comment:密码"`
    Nickname string `gorm:"type:varchar(100);comment:昵称"`
    Avatar   string `gorm:"type:varchar(255);default:'http://127.0.0.1:9001/browser/image/normal.png';comment:头像"`
    Role     string `gorm:"type:varchar(50);default:user;comment:角色"`
    Status   int    `gorm:"type:tinyint;default:1;comment:状态 1启用 0禁用"`
    Email    string `gorm:"type:varchar(100);not null;uniqueIndex;comment:邮箱"`
}
```

```go
type UserRole struct {
    UserID uint `gorm:"primaryKey;comment:用户ID"`
```

## 接口设计

所有公开接口不需要认证，用户自操作接口需 JWT，管理员接口需 JWT + admin 角色。

### 1. 获取验证码

```http
GET /api/captcha
```

返回：

```json
{
  "code": 200,
  "data": {
    "captcha_id": "uuid-string",
    "captcha_img": "data:image/png;base64,..."
  }
}
```

说明：

- 6 位数字验证码，存入 Redis（key=`captcha:<id>`，5 分钟过期）
- 前端需将 `captcha_id` 随登录/注册请求一起提交

### 2. 注册

```http
POST /api/register
Content-Type: application/json
```

Body：

```json
{
  "username": "newuser",
  "password": "Abc@1234",
  "email": "newuser@example.com",
  "nickname": "新用户",
  "captcha_id": "uuid-string",
  "captcha_code": "385210"
}
```

返回：

```json
{
  "code": 200,
  "msg": "注册成功",
  "data": {
    "id": 5,
    "username": "newuser",
    "nickname": "新用户",
    "email": "newuser@example.com",
    "role": "user",
    "status": 1
  }
}
```

说明：

- 默认角色为 `user`，默认状态为启用（status=1）
- 密码通过 bcrypt 加密存储
- 用户名和邮箱均做唯一性校验

### 3. 登录

```http
POST /api/login
Content-Type: application/json
```

Body：

```json
{
  "username": "admin",
  "password": "Admin@123",
  "captcha_id": "uuid-string",
  "captcha_code": "482917"
}
```

返回：

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "access_token": "eyJhbGciOi...",
    "refresh_token": "eyJhbGciOi...",
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "管理员",
      "avatar": "http://127.0.0.1:9001/browser/image/normal.png",
      "email": "admin@example.com",
      "role": "admin",
      "status": 1
    }
  }
}
```

说明：

- JWT 中包含 `userID`、`roles`（角色码数组）、`permissions`（权限码数组）
- 登录成功自动清除失败计数

### 4. 获取当前用户信息

```http
GET /api/user/info
Authorization: Bearer <token>
```

返回：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "nickname": "管理员",
    "avatar": "http://...",
    "email": "admin@example.com",
    "role": "admin",
    "status": 1
  }
}
```

### 5. 修改个人信息

```http
PUT /api/user/info
Content-Type: application/json
Authorization: Bearer <token>
```

Body：

```json
{
  "nickname": "新昵称",
  "email": "newemail@example.com"
}
```

说明：

- 只允许修改昵称、邮箱、头像
- 邮箱修改时做唯一性校验（排除自己）
- 不可修改用户名、角色、状态

### 6. 修改密码

```http
PUT /api/user/password
Content-Type: application/json
Authorization: Bearer <token>
```

Body：

```json
{
  "old_password": "OldPass@123",
  "new_password": "NewPass@456",
  "confirm_password": "NewPass@456"
}
```

说明：

- 需验证旧密码正确
- 新密码与确认密码必须一致
- 新密码不能与旧密码相同
- 新密码需满足复杂度要求

### 7. 上传头像

```http
POST /api/user/avatar
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

Body：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| file | file | 是 | 图片文件，最大 2MB |

约束：

- 仅支持 jpg / jpeg / png / webp 格式
- 上传到 MinIO `image` bucket，路径 `avatars/{userID}/`
- UUID 命名，保留最近 2 个旧头像文件

返回：

```json
{
  "code": 200,
  "msg": "头像上传成功",
  "data": {
    "id": 1,
    "username": "admin",
    "avatar": "http://127.0.0.1:9000/image/uuid-avatar.png"
  }
}
```

### 8. 登出

```http
POST /api/user/logout
Authorization: Bearer <token>
```

说明：

- 将当前 Token 加入 Redis 黑名单（key=`blacklist:<token>`，过期时间对齐 Token 有效期）
- JWT 中间件每次请求检查黑名单

### 9. 获取初始上下文

```http
GET /api/user/context
Authorization: Bearer <token>
```

返回：

```json
{
  "code": 200,
  "data": {
    "user": {
      "id": 1,
      "username": "admin",
      "nickname": "管理员",
      "avatar": "http://...",
      "email": "admin@example.com",
      "role": "admin",
      "status": 1
    },
    "roles": ["admin"],
    "permissions": ["user:list", "user:delete", "role:create"],
    "menus": [
      {
        "id": 1,
        "name": "系统管理",
        "path": "/system",
        "icon": "settings",
        "type": 1,
        "children": [...]
      }

```

说明：

- 一次性返回前端初始化所需的所有信息
- `roles` 和 `permissions` 从 JWT 中间件注入的 context 中读取
- `menus` 通过 `GetUserMenus()` 实时查询并组装树

- 不能操作自己和其他管理员

## 实现步骤

### 1. 创建模型

文件：

```text
model/user.go       → User 结构体

```

```text

```

### 2. 创建 DTO

文件：

```text

```

包含：

- `RegisterReq`、`LoginReq`
- `UserInfo`、`LoginResp`、`UserListResp`
- `UpdateSelfReq`、`ChangePasswordReq`
- `AdminUpdateUserReq`
- `InitialContextResp`

### 3. 创建 Service

文件：

```text
service/user.go         → Register, Login, UpdateSelf, ChangePassword, SetAvatar, GetUserInfo, Logout
service/user_helper.go   → validatePassword, isLocked, incrFailed, lockAfterFail, clearFailed
service/admin_user.go    → GetAllUsers, DeleteUserByAdmin, UpdateUserByAdmin, ToggleUserStatus
service/captcha.go       → GenerateCaptcha, VerifyCaptcha（Redis 存储适配）
```

### 4. 创建 Handler

文件：

```text
handler/user.go      → UserHandler（Register, Login, UpdateSelf, ChangePassword, Logout, UploadAvatar, GetInfo, InitialContext）
handler/admin_user.go → AdminUserHandler（ListUsers, UpdateUser, DeleteUser, ToggleStatus）
handler/captcha.go   → CaptchaHandler（Generate）
```

### 5. 更新路由

在 `router/router.go` 中：

```go
// 公开路由
api.GET("/captcha", captchaHandler.Generate)
api.POST("/register", userHandler.Register)
api.POST("/login", userHandler.Login)

// 需认证路由
user := api.Group("/user").Use(auth)
{
    user.GET("/context", userHandler.InitialContext)
    user.GET("/info", userHandler.GetInfo)
    user.PUT("/info", userHandler.UpdateSelf)
    user.PUT("/password", userHandler.ChangePassword)
    user.POST("/avatar", userHandler.UploadAvatar)
    user.POST("/logout", userHandler.Logout)
}

// 管理员路由
admin := api.Group("/admin").Use(auth, requireAdmin)
{
    admin.GET("/users", adminUserHandler.ListUsers)
    admin.PUT("/users/:id", adminUserHandler.UpdateUser)
    admin.DELETE("/users/:id", adminUserHandler.DeleteUser)
    admin.PUT("/users/:id/status", adminUserHandler.ToggleStatus)
}
```

## 数据流

### 注册流程

```text
用户提交注册表单
  → 验证码校验（Redis captcha:<id>）
  → 密码复杂度校验
  → 用户名/邮箱查重
  → bcrypt 加密密码
  → 写入 users 表（role=user, status=1）
  → 返回用户信息（脱敏）
```

### 登录流程

```text
用户提交登录表单
  → 验证码校验
  → 锁定检查（Redis lock:<username>）
  → 查找用户（按 username）
  → 状态检查（status=1）
  → bcrypt 密码比对
    ├─ 失败 → incr 失败计数 → 检查是否触发锁定
    └─ 成功 → 清除失败记录
              → 查询 user_roles → roles
              → 查询 role_permissions → permissions
              → 查询 role_menus → menus（初始上下文时）
              → 签发 JWT（含 userID + roles + permissions）
```

### 初始上下文数据流

```text
前端加载完成
  → GET /api/user/context
  → JWT 中间件注入 userID、roles、permissions 到 Gin Context
  → 查询用户信息
  → 查询用户菜单树（user → user_roles → role_menus → menus → 组装树）
  → 组装返回
```

## 安全约束

| 约束 | 说明 |
|---|---|
| 管理员不能操作自己 | DeleteUserByAdmin / UpdateUserByAdmin / ToggleUserStatus 均校验 |
| 管理员不能操作其他管理员 | 检查目标用户 role != "admin" |
| 密码永不明文存储 | bcrypt 加密，即使数据库泄露也无法还原 |
| Token 黑名单 | 登出后 Token 加入 Redis 黑名单，过期时间对齐 Token 有效期 |
| 验证码一次性消费 | 校验后立即删除（无论成功失败），防止重复提交和暴力穷举 |
| 邮箱唯一性 | 注册和修改均校验（排除自己） |

## Apifox 测试顺序

1. 获取验证码（GET /api/captcha）
2. 注册新用户（POST /api/register）
3. 获取验证码 → 登录（POST /api/login），拿到 access_token
4. 查个人信息（GET /api/user/info）
5. 修改个人信息（PUT /api/user/info）
6. 修改密码（PUT /api/user/password）
7. 上传头像（POST /api/user/avatar）
8. 获取初始上下文（GET /api/user/context）
9. 管理员登录 → 查用户列表（GET /api/admin/users）
10. 管理员修改用户（PUT /api/admin/users/:id）
11. 管理员 Ban/UnBan（PUT /api/admin/users/:id/status）
12. 管理员删除用户（DELETE /api/admin/users/:id）
13. 登出（POST /api/user/logout）
14. 用已登出的 Token 访问 → 验证黑名单生效

## 注意事项

- `UserInfo` 返回不包含 `Password` 字段，确保密码永不到达前端
- 头像上传使用 MinIO 直传，不经过应用服务器中转大文件
- `InitialContext` 接口中的 `roles` 和 `permissions` 从 JWT context 读取，不在接口内重复查库
- 管理员操作均有双重保护：不能操作自己 + 不能操作其他管理员
- 软删除用户后，其关联的 user_roles 不自动清理（保留历史关系）
- 密码修改后不强制登出（现有 Token 仍有效）
