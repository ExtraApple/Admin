# 数据权限实施文档

## 功能定位

数据权限用于控制不同管理员可以查看和操作哪些用户、组织数据。它不替代接口权限：接口权限决定“能不能访问接口”，数据权限决定“访问接口后能看见或操作哪些数据”。

## 数据范围

| data_scope | 含义 | 当前作用 |
|---|---|---|
| `all` | 全部数据 | 可查看和操作全部用户、全部组织 |
| `self` | 仅本人数据 | 用户列表只返回自己，组织列表为空 |
| `org` | 本组织数据 | 可查看和操作自己所属组织内的数据 |
| `org_and_children` | 本组织及下级组织数据 | 可查看和操作自己所属组织及下级组织内的数据 |
| `custom` | 自定义组织数据 | 根据 `role_data_scopes` 指定的组织过滤 |

超级管理员角色 `roles.code = admin` 固定拥有全部数据权限。

## 数据表

### roles.data_scope

角色表新增字段：

```text
data_scope varchar(50) default 'all'
```

### role_data_scopes

自定义组织数据范围关联表：

```text
role_id
organization_id
```

只有 `data_scope = custom` 时，该表才会参与过滤。

## 生效范围

当前已接入数据权限过滤的接口：

```http
GET    /api/admin/users
PUT    /api/admin/users/:id
DELETE /api/admin/users/:id
PUT    /api/admin/users/:id/status
PUT    /api/admin/users/:id/kick

GET    /api/admin/organizations
GET    /api/admin/organizations/tree
POST   /api/admin/organizations
PUT    /api/admin/organizations/:id
DELETE /api/admin/organizations/:id
POST   /api/admin/organizations/:id/users
GET    /api/admin/organizations/:id/users
```

## 配置角色数据权限

```http
POST /api/admin/roles/:id/data-scope
Authorization: Bearer <管理员token>
Content-Type: application/json
```

全部数据：

```json
{
  "data_scope": "all"
}
```

仅本人：

```json
{
  "data_scope": "self"
}
```

本组织：

```json
{
  "data_scope": "org"
}
```

本组织及下级组织：

```json
{
  "data_scope": "org_and_children"
}
```

自定义组织：

```json
{
  "data_scope": "custom",
  "organization_ids": [1, 2, 3]
}
```

成功返回：

```json
{
  "code": 200,
  "msg": "配置成功"
}
```

## 查询角色数据权限

```http
GET /api/admin/roles/:id/data-scope
Authorization: Bearer <管理员token>
```

成功返回：

```json
{
  "code": 200,
  "data": {
    "role_id": 2,
    "data_scope": "custom",
    "organization_ids": [1, 2, 3]
  }
}
```

## Apifox 测试流程

1. 用超级管理员登录，拿到 token。
2. 创建组织，例如总部、技术部、运营部。
3. 把测试用户绑定到组织。
4. 创建测试角色，例如 `dept_admin`。
5. 给测试角色分配访问用户列表、组织列表、组织树的接口权限。
6. 调用 `POST /api/admin/roles/:id/data-scope` 设置数据范围。
7. 把测试用户绑定到该角色。
8. 用测试用户重新登录，拿到新 token。
9. 调用 `GET /api/admin/users`、`GET /api/admin/organizations`、`GET /api/admin/organizations/tree` 验证只能看到允许范围内的数据。

## 注意事项

- 修改角色数据权限后，会提升该角色下用户的 `token_version`，旧 token 会失效，需要重新登录。
- 新增的两个角色数据权限接口需要同步到 API 权限体系：

```http
POST /api/admin/apis/sync
POST /api/admin/apis/sync-permissions
```

- 如果普通管理员没有对应 API 权限，即使数据范围允许，也不能访问接口。
- 如果角色是 `admin`，系统固定视为全部数据权限，不允许通过数据权限接口修改。
